1. Visão Geral do Projeto

O PSPart-Website é um site institucional com e-commerce integrado para venda de peças e componentes de automação. A arquitetura foi projetada para funcionar em hospedagem compartilhada de baixo custo, sem dependências complexas.

ItemDetalhe
TipoSite estático + backend PHP com API REST
FrontendHTML / CSS / JavaScript — Bootstrap 5.3, Font Awesome, AOS
BackendPHP puro (sem framework)
Banco de dadosSQLite (arquivo .db, compatível com hospedagem compartilhada)
PagamentosMercado Pago — Checkout Pro (redirect)
Formulário de contatoFormSubmit.co → filipe@pentasis.com.br
AnalyticsGoogle Analytics GA4 com Consent Mode v2 (LGPD)
CEP automáticoViaCEP (gratuito, sem cadastro)
Hospedagem previstaHospedagem compartilhada com suporte a PHP e SQLite

5. Roadmap Futuro (Médio / Longo Prazo)

Funcionalidades que aumentam a escalabilidade e a competitividade, mas exigem mais investimento.

5.1 Logística e Frete

OpçãoDescriçãoCusto
Frete manual (atual) Combinado por WhatsApp/e-mail após a compra — simples e sem integração Grátis
Melhor Envio API gratuita — calcula frete em tempo real (Correios + transportadoras), gera etiqueta automaticamente. Recomendado quando o volume de pedidos crescer. Grátis (comissão por envio)
Correios API Integração direta com SEDEX/PAC para cálculo de frete no checkout Grátis (requer contrato com Correios)
💡 O Mercado Envios (Mercado Livre) não está disponível para lojas externas — funciona apenas dentro do marketplace do ML.

5.4 Infraestrutura

RecursoBenefícioQuando considerar
Migrar SQLite → MySQL/PostgreSQL Melhor performance e concorrência em alto volume Quando ultrapassar ~500 pedidos/mês
CDN para imagens Carregamento mais rápido das imagens dos produtos Quando o banco de imagens crescer
Hospedagem VPS/Cloud Mais controle, performance e segurança Quando o volume justificar o custo
Mercado Pago Checkout Transparente (Bricks) Comprador não sai do site para pagar — melhor conversão Quando houver equipe/recurso para manutenção contínua
